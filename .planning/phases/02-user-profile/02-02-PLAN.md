**1. Create `AvatarDisplay` component** at `apps/native/src/components/AvatarDisplay.tsx`:

Per D-08, users without an avatar see initials on a colored background.

Props: `{ uri: string | null; username: string; size: number }`.

```typescript
import React from "react";
import { View, Text, Image, StyleSheet } from "react-native";

const PALETTE = ["#E57373","#64B5F6","#81C784","#FFB74D","#BA68C8","#4DB6AC"];

function avatarColor(username: string): string {
  const hash = username.split("").reduce((acc, c) => acc + c.charCodeAt(0), 0);
  return PALETTE[hash % PALETTE.length];
}

function getInitials(username: string): string {
  return username.slice(0, 2).toUpperCase();
}

type AvatarDisplayProps = {
  uri: string | null;
  username: string;
  size: number;
};

const AvatarDisplay: React.FC<AvatarDisplayProps> = ({ uri, username, size }) => {
  if (uri) {
    return (
      <Image
        source={{ uri }}
        style={{ width: size, height: size, borderRadius: size / 2 }}
      />
    );
  }
  return (
    <View style={[{
      width: size, height: size, borderRadius: size / 2,
      backgroundColor: avatarColor(username),
      alignItems: "center", justifyContent: "center",
    }]}>
      <Text style={{ color: "#FFFFFF", fontSize: size * 0.4, fontWeight: "700" }}>
        {getInitials(username)}
      </Text>
    </View>
  );
};

export default AvatarDisplay;
```

Per D-03, this is a dedicated edit form.

Fields:
- **Avatar**: Tappable `AvatarDisplay`. On tap, show a simple `Modal` bottom sheet (per research open question 2 — use Modal, not a heavy library) with two options: "Choose from Library" and "Take Photo". Per D-05 and D-06:
  - "Choose from Library": call `ImagePicker.launchImageLibraryAsync({ mediaTypes: ["images"], allowsEditing: true, aspect: [1, 1], quality: 0.8 })`.
  - "Take Photo": call `ImagePicker.launchCameraAsync({ mediaTypes: ["images"], allowsEditing: true, aspect: [1, 1], quality: 0.8 })`.
  - After picking, resize to 400x400 using `ImageManipulator.manipulate(uri)` context builder: `.resize({ width: 400, height: 400 })`, `.renderAsync()`, `.saveAsync({ format: SaveFormat.JPEG, compress: 0.8 })`.
  - Get presigned URL: POST /me/avatar-upload-url.
  - Upload to R2: `fetch(uploadUrl, { method: "PUT", body: { uri: croppedUri, type: "image/jpeg", name: "avatar.jpg" } as any, headers: { "Content-Type": "image/jpeg" } })`.
  - Save public URL: PATCH /me with `{ avatarUrl: publicUrl }`.
  - Update local state to show new avatar immediately.

- **Username** (per D-09): Text input, pre-filled with current username. Min 3, max 30 chars. Per D-14, show character count.

- **Experience Level** (per D-16): Three-option segmented control or radio group for BEGINNER / INTERMEDIATE / ADVANCED. Pre-select current value.

- **Goal** (per D-12):
  - `goalType` picker: dropdown or segmented control with SKILL / STRENGTH / ENDURANCE / MOBILITY / CONSISTENCY.

  - Optional `targetValue` numeric input and `targetUnit` text input (per D-13).

- **Save button**: Sends PATCH /me for username + experienceLevel, POST /me/goals for goal if changed. On success, navigate back to profile screen. On 409 (duplicate username), show error message on the username field.

- **Cancel button**: Navigate back without saving.

- Use `useTheme()` for all colors. Use `ScrollView` + `SafeAreaView` for the form.
